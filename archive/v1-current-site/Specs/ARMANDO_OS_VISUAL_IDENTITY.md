# ARMANDO_OS_VISUAL_IDENTITY.md

## Visual Identity & Design System

------------------------------------------------------------------------

# 1. Core Design Principle

ArmandoOS should feel:

-   Controlled
-   Intentional
-   Systemic
-   Slightly futuristic
-   Mature, not trendy

If something feels loud, remove it.\
If something feels cluttered, simplify it.\
The power is in restraint.

------------------------------------------------------------------------

# 2. Visual Personality Archetype

Blend of:

-   NASA mission control
-   High-end developer tool
-   Subtle sci-fi UI
-   Executive dashboard

NOT:

-   Gaming UI
-   Hacker neon overload
-   Heavy gradients
-   Corporate template

------------------------------------------------------------------------

# 3. Color System

## 3.1 Base Palette

Primary Background:\
Very dark slate, not pure black.

Secondary Background:\
Slightly lighter surface for cards and modules.

Text Primary:\
Soft off-white.

Text Secondary:\
Muted gray.

Accent Primary (default theme):\
Electric blue.

Accent Secondary:\
Warm amber for highlight moments.

Completion State Color:\
Soft emerald glow.

------------------------------------------------------------------------

## 3.2 Operator Mode Theme

When unlocked:

Accent shifts to: - Deep cyan - Subtle grid glow - Slightly increased
contrast

Background slightly darkens.

No rainbow effects.\
Just refined system energy.

------------------------------------------------------------------------

# 4. Typography System

## 4.1 Font Strategy

Use two font families:

Primary: Modern sans-serif\
Clean, minimal\
Slightly technical

Secondary (Optional Accent): Monospace for: - XP meter - Terminal mode -
Simulator output

Do not overuse monospace.

------------------------------------------------------------------------

## 4.2 Type Scale

Large Headline: Used only on homepage core node.

Section Headers: Strong but not oversized.

Body: Readable, not tiny.

Microcopy: Slightly muted gray.

Whitespace matters more than font size.

------------------------------------------------------------------------

# 5. Layout Principles

## 5.1 Spacing

Use generous spacing.\
Modules should breathe.

Maintain a consistent spacing scale across the app.

Never stack dense paragraphs.

------------------------------------------------------------------------

## 5.2 Grid System

Invisible grid overlay on homepage.

Very subtle opacity.\
User should feel structure without consciously seeing it.

------------------------------------------------------------------------

# 6. Motion System

Motion should feel:

-   Smooth
-   Intentional
-   Calm

Never bouncy.\
Never exaggerated.

------------------------------------------------------------------------

## 6.1 Animation Rules

Hover: Small scale increase\
Soft glow\
150--250ms

Page transitions: Fade + slight zoom\
No slide carousels

XP toast: Quick appear\
Subtle upward drift\
Fade out

Orbit movement: Very slow drift\
No perfect symmetry

------------------------------------------------------------------------

# 7. Iconography

Icons should be:

-   Minimal
-   Line-based
-   Consistent stroke width

Avoid filled cartoon icons.

Prefer system-like glyphs.

------------------------------------------------------------------------

# 8. Module Visual Treatment

Each ModuleNode:

-   Circular
-   Thin outer ring
-   Soft glow on hover
-   Inner icon + label

Completed modules: Subtle green ring accent

Unlocked modules: Accent glow

Locked modules: Dimmed opacity

------------------------------------------------------------------------

# 9. Card System (Projects + Loyalty)

Cards should:

-   Have soft rounded corners
-   Subtle shadow
-   No heavy border

Flip animation: Smooth rotation\
No dramatic spin

Back side: Muted background\
Focused content

------------------------------------------------------------------------

# 10. Systems Graph Visual Rules

Nodes: - Small circular anchor - Expands into panel - Dim other nodes
when active

Lines: - Thin - Subtle glow when active

Must feel readable and structured.

------------------------------------------------------------------------

# 11. Signal Accent Mode (Music Layer)

Tone toggles change:

-   Accent color
-   Background pulse speed
-   Micro motion intensity

Clean: Cool blue

Crunch: Slight amber warmth

Metal: Deeper blue + subtle intensity bump

No flashing.\
Refined energy only.

------------------------------------------------------------------------

# 12. Constellation Section

Stars:

-   Small glowing dots
-   Connect with thin lines
-   Hover increases brightness

When 4+ hovered: Subtle shimmer effect across constellation.

Should feel poetic, not childish.

------------------------------------------------------------------------

# 13. Microcopy Tone

Language style:

-   Calm
-   Reflective
-   Direct
-   Structured

Avoid buzzwords such as: - Passionate - Results-driven - Rockstar

Focus on clarity and specific outcomes.

------------------------------------------------------------------------

# 14. Accessibility Rules

-   Maintain strong contrast ratios
-   Avoid tiny text
-   Ensure hover states have keyboard equivalents
-   Do not rely on color alone for meaning

------------------------------------------------------------------------

# 15. What To Avoid

-   Neon gradients
-   Excessive motion
-   Too many colors
-   Cluttered hero section
-   Long paragraphs
-   Corporate stock photography

No photos required.\
The interface is the personality.

------------------------------------------------------------------------

# 16. The Feeling Test

When loading the homepage, ask:

Does this feel like: - A system? - A mind at work? - Controlled power?

If yes, keep it.\
If not, refine until it does.
